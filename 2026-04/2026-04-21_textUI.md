## 2️⃣ `Texts` コントローラーに `show` アクションを追加

```ruby
class TextsController < ApplicationController
  # ログインユーザーにテキストを表示するためのコントローラー
  before_action :require_login
  before_action :set_text
  before_action :set_reading_session
  before_action :require_membership


  def show
    session_text = @reading_session.tests.order(:id) # #95 で position カラムを追加したら order(:position) に変更
    current_index = session_text.index(@text)
    @prev_text = session_text[current_index - 1] if current_index > 0
    @next_text = session_text[current_index + 1]
  end

  private

  # テキストをセットするためのコールバック
  def set_text
    @text = Text.find(params[:id])
  end

  # リーディングセッションをセットするためのコールバック
  def set_reading_session
    @reading_session = ReadingSession.find(params[:reading_session_id])
  end

  # セッションへの参加を確認するためのコールバック
  def require_membership
    unless @reading_session.members.include?(current_user)
      redirect_to reading_sessions_path, alert: "参加していないセッションです"
    end
  end
end
```

## ブラウザの `NoMethodError` に対応

- `has_many :users` → **残す**（Rails 標準、データ層）
- `has_many :members, source: :user` → **追加**（ドメイン層）
- コントローラーの参加チェックは `.members` に統一

```ruby
# app/models/reading_session.rb
has_many :memberships, dependent: :destroy
has_many :users, through: :memberships   # ← 既存のまま残してもOK、または削除
has_many :members, through: :memberships, source: :user  # ← 追加
```

> **Rails TIPS**

### アソシエーション命名のガイドライン

> 原則

| 観点 | 命名 | 例 |
| --- | --- | --- |
| **データ層**（モデル・DB の関係を表す） | モデル名に合わせる | `has_many :users` |
| **ドメイン層**（業務的な意味を表す） | ドキュメントの用語に合わせる | `has_many :members` |

> この app での判断

- `ReadingSession` にとって `users` は「参加しているユーザー」という**データ上の関係**であり、`members` は「セッションの参加者」という**ドメイン上の役割**。
- どちらも正当な理由があるので、**両方定義して使い分ける**のが現実的。

```ruby
# reading_session.rb
has_many :users, through: :memberships          # データ層（Rails 標準の形）
has_many :members, through: :memberships,       # ドメイン層（業務的な意味）
         source: :user
```

> 使い分けの基準

```ruby
# ✅ .users → データを扱うとき（join, eager load など）
ReadingSession.includes(:users)

# ✅ .members → 業務ロジックを書くとき（参加チェックなど）
@reading_session.members.include?(current_user)
```
