# [co-READER](https://github.com/QynToKey/co_reader)（day: 17_1）： `user.name` カラムの実装漏れ対応

## 0️⃣ 現状と対策

> 現状

ER 図では `users.name` を NOT NULL として定義しているが、マイグレーション・バリデーション・フォームのいずれにも実装されていない。

> 対策

Render 無料枠ではコンソール接続不可のため、マイグレーション①で既存レコードを `default: "未設定"` で自動的に処理し、開発者自身の name を登録後、マイグレーション②で default を削除。

---

## 1️⃣ マイグレーションを作成 / 実行

```bash
$ docker compose exec web bin/rails g migration AddNameToUsers name:string
      invoke  active_record
      create    db/migrate/20260424042707_add_name_to_users.rb
```

  ⬇️

```ruby
# db/migrate/20260424042707_add_name_to_users.rb
class AddNameToUsers < ActiveRecord::Migration[8.1]
  def change
    add_column :users, :name, :string, null: false, default: "未設定"
  end
end
```

👉 *バリデーションエラーを回避するため `default: "未設定"` を追記*

  ⬇️

```bash
$ docker compose exec web bin/rails db:migrate
```

## 4️⃣ `admin/members` に name を追加

### `Admin::MembersController`

```ruby
# app/controllers/admin/members_controller.rb
  def user_params
    params.require(:user).permit(:name, :email, :password, :password_confirmation) # ⬅️ :name を追加
  end

  def update_params
    params.require(:user).permit(:name, :email, :password, :password_confirmation).reject do |_, v| # ⬅️ :name を追加

    end
  end
```
