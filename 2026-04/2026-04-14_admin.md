# [co-READER](https://github.com/QynToKey/co_reader)（day: 8）： 管理者アカウント登録フロー

## 0️⃣ 現状と実装方針

### 現状

```ruby
# app/controllers/users_controller.rb
  def create
    @user = User.new(user_params) # ⬅️ user_params は email, password, password_confirmation のみ
    ・・・
  end
```

👉 *`role` が `params` に含まれないため、DB のデフォルト値 0（= member）で作成される。つまり 現状は誰でも `/register` から一般ユーザー（member）として登録できる 状態*

### 実装方針

- admin ロール付きアカウントを、 `/register` からではなく、使い捨てトークン付き URL `/admin/register?token=xxx` 経由でのみ作成可能にする。
- トークンは `seed` で生成し、使用後に無効化する。
- 現状の `/register` は member アカウント専用のまま残す。
