# [co-READER](https://github.com/QynToKey/co_reader)（day: 6）： i18n の設定

## 0️⃣ 背景

- Bootstrap 導入後、ビュー実装が本格化する前に i18n を設定する。
- `rails-i18n gem` 未導入・`default_locale` 未設定の状態。
- バリデーションエラーが英語で出力されているため、日本語化が必要。

> 📝 `i18n` ：
Internationalization（国際化）の略。

---

## 1️⃣ Gemfile に `rails-i18n` を追加

```ruby
# Gemfile
gem "rails-i18n"
```

  ⬇️

```bash
docker compose exec web bundle install
```

---

## 2️⃣ `config/application.rb` に `locale` 設定を追加

👉 *Rails に「このアプリのデフォルト言語は日本語（:ja）だ」と伝える*

```ruby
# config/application.rb
    config.i18n.default_locale = :ja
```

- この実装によって、Rails が内部で言語ファイルを探すときに ja のものを優先して使うようになる。
- `rails-i18n gem` がその ja 用の翻訳ファイルを持っているため、バリデーションエラー等が日本語になる。 *（設定しなかった場合、Rails のデフォルトは :en（英語）なので、バリデーションエラーは `can't be blank` のような英語になる）*

> 📝 `:ja` ：
[IETF言語タグ](https://ja.wikipedia.org/wiki/IETF%E8%A8%80%E8%AA%9E%E3%82%BF%E3%82%B0)のルールに基づいた識別子。
日本語は ja、英語は en、中国語は zhなど。
Ruby ではシンボル（`:` 付き）で表現する。

---

## 3️⃣ `config/locales/en.yml` を削除

```bash
rm config/locales/en.yml
```

> 📝 **削除する理由**：

1. 内容が不要 — 現在の `en.yml` には `hello: "Hello world"` しかなく、アプリで使っていない
2. 英語ロケールを使わない *（日本語専用アプリなので en の翻訳ファイル自体が不要）*

## 4️⃣ `activerecord/ja.yml` / `views/ja.yml` を作成

```bash
config/locales
├── activerecord
│   └── ja.yml
├── ja.yml
└── views
    └── ja.yml
```

```ruby
# activerecord/ja.yml
ja:
  activerecord:
    attributes:
      user:
        email: "メールアドレス"
        password: "パスワード"
        password_confirmation: "パスワード（確認）"
```

```ruby
# views/ja.yml
ja:
  layouts:
    application:

  sessions:

      title: "ログイン"

```

📝 `human_attribute_name` ：
Rails: モデル非バインドフォームで AR の属性名翻訳を再利用する

- `form_with url:` はモデルと紐づかないため、`f.label :email` だけでは `activerecord.attributes.user.email` を参照できず、英語のまま表示される。
- `Model.human_attribute_name(:attr)` を使うと、 `activerecord/ja.yml` に定義した翻訳を明示的に取得できる。

```erb
<%= f.label :email, User.human_attribute_name(:email), class: "form-label" %>
```

👉 *これにより `activerecord/ja.yml`に一元化した翻訳を重複定義なしで参照できる*

> 使い分け

| フォームの種類 | ラベルの書き方 |
| --- | --- |
| `form_with model:` | `f.label :email`（AR ルックアップ自動） |
| `form_with url:` | `f.label :email, User.human_attribute_name(:email)` |
