```bash
touch app/controllers/texts_controller.rb
```

```ruby
# app/controllers/texts_controller.rb
class TextsController < ApplicationController
  # ログインユーザーにテキストを表示するためのコントローラー
  before_action :require_login
  before_action :set_text

  def show
  end

  private

  # コールバックを使用して、共通のセットアップまたは制約をまとめる。
  def set_text
    @text = Text.find(params[:id])
  end
end
```

### Git TIPS

#### 自動クローズキーワード

GitHub で、 issue を自動クローズする公式キーワード。

| キーワード |
| --- |
| `close` / `closes` / `closed` |
| `fix` / `fixes` / `fixed` |
| `resolve` / `resolves` / `resolved` |

> 使い方

- 上記の「キーワード + issue 番号」で機能する。*（大文字・小文字は問わない）*
- PR の本文（description）に書く。*（コミットメッセージやコメントには書かない。本文のみ有効）*
- `#番号` の形式で書く

```text
Closes #13
```

- 複数 issue を閉じる場合はカンマまたは改行で並べる

```test
Closes #13, Closes #14
```

> 使い分け

意味として自然なものを選べばOKで、機能は同じ。

- バグ修正 → `Fixes #番号`
- 機能実装 → `Closes #番号`
- タスク消化 → `Resolves #番号`
