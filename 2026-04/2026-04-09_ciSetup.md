## CI の目的

コードを push / PR するたびに、以下を自動でチェックする仕組み。

| ジョブ | 内容 |
| --- | --- |
| `scan_ruby` | Rails の脆弱性パターン検出（Brakeman）、gem の既知脆弱性（bundler-audit） |
| `scan_js` | JS 依存パッケージの脆弱性 |
| `lint` | RuboCop によるコードスタイルチェック |
| `test` | minitest によるテスト自動実行（今回追加） |

CI が保証するのは「書かれたテストが通ること」。テストの内容・十分さは開発者が担保する。

## 修正内容

### 1. `actions/checkout@v6` → `@v4`

`v6` は存在しないバージョン。全3ジョブで同じ誤りがあったので修正した。

### 2. `test` ジョブを追加

PostgreSQL が必要なため GitHub Actions の **service container** を使う。

```yaml
services:
  postgres:
    image: postgres:16
    env:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    options: >-
      --health-cmd pg_isready
      --health-interval 10s
      --health-timeout 5s
      --health-retries 5
    ports:
      - 5432:5432
```

CI 環境では UNIX ソケットが使えないため `DATABASE_URL` で TCP 接続を指定する。

```yaml
env:
  RAILS_ENV: test
  DATABASE_URL: postgres://postgres:postgres@localhost:5432/app_test
```

### 3. `db:schema:load` → `db:prepare`

最初 `db:create db:schema:load` と書いたが、`schema.rb` がまだ存在しないため CI が失敗した。

`db:prepare` は状況を自動判断してくれる：

- `schema.rb` あり → `schema:load`（高速）
- `schema.rb` なし → `db:migrate` にフォールバック

DB 作成も内包しているため `db:create` も不要になる。
