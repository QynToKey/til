## 1️⃣ `reading_sessions` テーブル作成マイグレーション

```bash
$ docker compose exec web bin/rails generate migration CreateReadingSessions \
  text:references \
  created_by:integer \
  name:string \
  invite_token:string
      invoke  active_record
      create    db/migrate/20260419032335_create_reading_sessions.rb
```

  ⬇️

```ruby
# db/migrate/20260419032335_create_reading_sessions.rb
class CreateReadingSessions < ActiveRecord::Migration[8.1]
  def change
    create_table :reading_sessions do |t|
      t.references :text, null: false, foreign_key: true
      # セッション作成者を users テーブルの id として参照するため、foreign_key オプションで to_table を指定
      t.references :created_by, null: false, foreign_key: { to_table: :users }
      t.string :name
      t.string :invite_token, null: false

      t.timestamps
    end

    # invite_token にユニークインデックスを追加して、重複を防止
    add_index :reading_sessions, :invite_token, unique: true
  end
end
```

👉 *`created_by` を `t.integer` → `t.references` に変更（FK 制約と index が自動で付く）*
👉 *`invite_token` に `null: false` を追加*
👉 *`invite_token` に `UNIQUE index` を追加*

## 2️⃣ `membership` テーブル作成マイグレーション

```bash
$ docker compose exec web bin/rails generate migration CreateMemberships
      invoke  active_record
      create    db/migrate/20260419040404_create_memberships.rb
```

```ruby
# db/migrate/20260419040404_create_memberships.rb
class CreateMemberships < ActiveRecord::Migration[8.1]
  def change
    create_table :memberships do |t|
      t.references :user, null: false, foreign_key: true
      t.references :reading_session, null: false, foreign_key: true
      t.integer :session_role, null: false, default: 0 # 0: member, 1: session_admin
      t.integer :mode, null: false, default: 0 # 0: solo, 1: 招待済み, 2: co_reading
      t.timestamps
    end

    add_index :memberships, [:user_id, :reading_session_id], unique: true # 同じユーザーが同じセッションに複数回参加するのを防止
  end
end
```

👉 *`session_role` と `mode` は `enum` 用に `integer` で定義（Rails の `enum` はデフォルト値必須）*
👉 *`user_id` + `reading_session_id` の「複合 UNIQUE index」で同一セッションへの重複参加を防ぐ*
👉 *`mode` は #70 の設計メモにはないが、ER 図上 `memberships.mode` が必要なため含めた*

  ⬇️
