# [co-READER](https://github.com/QynToKey/co_reader)（day: 7）： ユーザーロール管理

## 0️⃣ 現状と実装方針

- 現状の認証システム（Sorcery）は「ログイン済み/未ログイン」の二択のみ。
- 管理者と参加者を区別するロール機能を追加し、将来の機能（テキスト管理・グループ管理等）に権限ベースのアクセス制御を付加する。

| ファイル | 変更種別 |
| --- | --- |
| `db/migrate/TIMESTAMP_add_role_to_users.rb` | 新規作成 |
| `app/models/user.rb` | enum + validates 追加 |
| `app/controllers/application_controller.rb` | `require_admin` 追加 |
| `config/locales/ja.yml` | エラーメッセージ追加 |
