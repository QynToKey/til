## [卒制](https://github.com/QynToKey/HowLongWillItLast)のための`README`提出など

### 何が起きたか

- `README` を **main ブランチ上で先に書いてしまった**
- その後でブランチを切って `PR` を作成
- 講師から「差分がありません」と却下された

#### 求められた指示（リカバリーのため）

1. 本プルリクエストをクローズする。
2. `git add .` を実行。
3. `git commit -m commit` を実行。
4. (ローカル環境で)`git checkout main` を実行してmainブランチへ遷移。
5. `git branch`を実行。（mainブランチに居ることを確認する）
6. `git pull origin main`
7. `README.md`を白紙状態にして保存。
8. `git add README.md` を実行。
9. `git commit -m 'reset README.md'` を実行。
10. `git push origin main` を実行。
11. `git checkout -b readme_review_request_2 `を実行。
12. `git branch` を実行。（`readme_review_request_2`ブランチに居ることを確認する）
13. `README.md`を編集、コミット、プッシュ、プルリクエスト作成。
14. 作成したプルリクエストのURLを提出。（以降、READMEレビューに関しては、提出したプルリクエスト上でやり取りを行う）

  ⚠️ *この手順は「差分を強制的に作るためのリカバリー手段」であり、本来は `main` ブランチで `README` を消す操作は行うべきではない*


#### 正しい手順（本来こうするべきだった）

1. **main では作業しない**
2. 先にブランチを切る

   ```bash
   git checkout -b docs/readme
   ```
3. ブランチ上で `README` を作成・編集
4. `commit` & `push`

   ```bash
   git add README.md
   git commit -m "docs: add initial README"
   git push origin docs/readme
   ```
5. main ← docs/readme の PR を作成

👉 *`PR` は「ブランチ間の差分」を出す仕組み*

---

### 学んだこと（重要）

-  `PR` は**コミットの有無**ではなく「**差分**」を示す仕組み
- ブランチは「作業前」に切る
- ブランチを削除してもコミットは消えない
  ```bash
  # マージしていないブランチを削除
    git branch -D <branch>

  # ブランチの削除をリモートへpush
    git push origin --delete <branch>
  ```

---

### 今後の自分ルール

- 何か提出する作業は **必ず先にブランチを切る** <br>👉 *`main `は「触らない前提」で扱う*
- 使い終わったブランチはローカル・リモート両方削除
- `README` やドキュメント変更は `docs:` プレフィックス([接頭辞](https://qiita.com/muranakar/items/20a7927ffa63a5ca226a#%E8%A1%A8))を使う
  | 主な接頭辞 | 意味 |
  |:---:| --- |
  | `feat:` | 新しい機能の追加 |
  | `fix:` | バグ修正 |
  | `docs:` | ドキュメント（`README` など） |
  | `clean:` / `refactor:` | リファクタ |
  | `chore:` | 上に当てはまらない雑務や設定など |

  👉 *これらに、`add` `update` `delete` などを必要に応じてコミット本文に添える*


---

### Rulesets（GitHub）についてのメモ（次に学ぶこと）

- `Rulesets` は「**特定ブランチを事故から守る**ためのルール」
- `public` リポジトリでは「`main` ブランチ保護」が前提
- 主な目的：
  - main への**直接`push`**を防ぐ
  - **誤削除**や`force push` を防ぐ
  - **`PR` 経由**でのみ変更を入れる

  👉 *当面は private リポジトリで PR 操作に慣れてから public リポジトリに適用したい*
