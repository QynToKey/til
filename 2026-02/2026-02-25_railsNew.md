## 2️⃣ Docker 環境構築

### Dockerfile.dev 作成

#### 設計方針

- `Ruby 3.2.3` を使用
- 不要な `Node` / `Yarn` は導入しない *（素のJavaScriptを使用予定）*
- 作業ディレクトリを `/how_long_will_it_last` に統一

```bash
FROM ruby:3.2.3

ENV LANG=C.UTF-8
ENV TZ=Asia/Tokyo

RUN apt-get update -qq && \
    apt-get install -y build-essential libpq-dev vim postgresql-client

WORKDIR /how_long_will_it_last

COPY . /how_long_will_it_last

RUN bundle install
```

---

### compose.yml 作成

```bash
services:
  db:
    image: postgres:15   # バージョン変更のリスク対応のため明示的に指定
    restart: always
    environment:
      POSTGRES_PASSWORD: password
      TZ: Asia/Tokyo
    volumes:
      - postgresql_data:/var/lib/postgresql/data   # data の保存場所を指定
    ports:
      - "5432:5432"
    healthcheck:   # ビルド後に追加
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5

  web:
    build:
      context: .
      dockerfile: Dockerfile.dev
    command: bash -c "bundle install && bundle exec rails db:prepare && rm -f tmp/pids/server.pid && bin/rails server -b 0.0.0.0"   # ビルド後にコマンドを更新
    volumes:
      - .:/how_long_will_it_last
      - bundle_data:/usr/local/bundle
    ports:
      - "3000:3000"
    depends_on:   # ビルド後に healthcheck 連動に変更
      db:
        condition: service_healthy
    stdin_open: true
    tty: true
    environment:
      TZ: Asia/Tokyo

volumes:
  postgresql_data:
  bundle_data:
```

👉 *「PostgreSQL環境」を用意し、データ永続化を行った*
👉 *ビルド後に `bundle exec rails db:prepare` を起動コマンドに加えたことによって、`docker compose up` 実行時に「DB接続」および「接続確認」が自動実行される構成とした*

#### db 接続設定

`config/database.yml` を修正：

```bash
default: &default
  adapter: postgresql
  encoding: unicode
  pool: <%= ENV.fetch("RAILS_MAX_THREADS") { 5 } %>
  host: db
  username: postgres
  password: password
```

👉 *`Docker Compose` では `services.db` の名前がホスト名になるため、`localhost` ではなく `db` を指定する必要がある*

## 4️⃣ DB接続エラーの修正

### エラー

```bash
FATAL: database "how_long_will_it_last_development" does not exist
```

### 原因

- コンテナ間の通信設定が不足していた
- DB 作成前に接続を試行した

### 対応

- `database.yml` に `host: db` 追加
- `healthcheck` を「DB存在確認」ではなく「PostgreSQL起動確認」に修正
