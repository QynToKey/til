## 2️⃣ Docker 環境構築

### Dockerfile.dev 作成

- ベースイメージ：`Ruby 3.2.3`
- 必要最小限のパッケージのみインストール

```bash
FROM ruby:3.2.3

ENV LANG=C.UTF-8
ENV TZ=Asia/Tokyo

RUN apt-get update -qq && \
    apt-get install -y build-essential libpq-dev vim postgresql-client

WORKDIR /how_long_will_it_last

COPY . /how_long_will_it_last

RUN bundle install
```

  👉 *作業ディレクトリを `/how_long_will_it_last` に設定*
  👉 *`Node` / `Yarn` は導入せず（素のJSを使用予定のため）*

---

### compose.yml 作成

```bash
services:
  db:
    image: postgres:15   # バージョン変更のリスク対応のため明示的に指定
    restart: always
    environment:
      POSTGRES_PASSWORD: password
      TZ: Asia/Tokyo
    volumes:
      - postgresql_data:/var/lib/postgresql/data   # data の保存場所を指定
    ports:
      - "5432:5432"
    healthcheck:   # ビルド後に追加
      test: ["CMD-SHELL", "pg_isready -d how_long_will_it_last_development -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5

  web:
    build:
      context: .
      dockerfile: Dockerfile.dev
    command: bash -c "bundle install && bundle exec rails db:prepare && rm -f tmp/pids/server.pid && bin/rails server -b 0.0.0.0"   # ビルド後にコマンドを更新
    volumes:
      - .:/how_long_will_it_last
      - bundle_data:/usr/local/bundle
    ports:
      - "3000:3000"
    depends_on:   # ビルド後に healthcheck 連動に変更
      db:
        condition: service_healthy
    stdin_open: true
    tty: true
    environment:
      TZ: Asia/Tokyo

volumes:
  postgresql_data:
  bundle_data:
```

#### db サービス

- `postgres:15`を使用
- `volume` でデータ永続化
- `healthcheck` で `pg_isready` を使用

#### web サービス

- bundle install
- rails db:prepare
- Rails server 起動

コンテナ間通信のため：

- `database.yml` に `host: db` を設定

  ```bash
  default: &default
    adapter: postgresql
    encoding: unicode
    pool: <%= ENV.fetch("RAILS_MAX_THREADS") { 5 } %>
    host: db
    username: postgres
    password: password
  ```

## 3️⃣ Rails アプリ生成 `rails new`

```bash
docker compose run web rails new . --force --database=postgresql
```

- `Rails 7.2.x` （7系の最新版）
- `Gemfile` 自動生成
- `PostgreSQL` 設定

## 4️⃣ DB接続エラーの修正

発生したエラー：

```bash
FATAL: database "how_long_will_it_last_development" does not exist
```

対応：

- healthcheck の見直し
- `database.yml` に `host: db` 追加
