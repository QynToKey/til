## RESTful について

### REST: REpresentational State Transfer

- Webアプリで**リソース**を**HTTPメソッド**で操作する設計思想
- Railsでは `resources` を使うことで自動的に`RESTfulルート`を生成する
  <br>👉 具体的には、
  - **データベース**における４つの基本的な操作「作成`Create`・取得 `Read`・更新 (Update)・削除 (Delete)」と、
  - **HTTP**における4つの基本的な操作「作成 (POST)・取得 (GET)・更新 (PATCH)・削除 (DELETE)」を<br>それぞれ繋げる

  | HTTP Method | Path | Controller#Action | CRUD |
  | --- | --- | --- | --- |
  | GET | /posts | posts#index | Read |
  | GET | /posts/new | posts#new | Create |
  | POST | /posts | posts#create | Create |
  | GET | /posts/:id | posts#show | Read |
  | GET | /posts/:id/edit | posts#edit | Update |
  | PATCH or PUT | /posts/:id | posts#update | Update |
  | DELETE | /posts/:id | posts#destroy | Delete|

### 「[実世界の複雑さに対するRESTfulな考え方を学べる講演動画](https://kaigionrails.org/2023/talks/moro/)」より

- 世の中の「複雑なふるまい」(= **動詞**)を**名詞に変換する**、と考えてみる。
  <br>👉 *例: 「ポストする」⇨「ポストを create する」*

- **イベントエンティティ**（Railsでは「中間モデル」として実装されることが多い）を「名詞／コト」として見出せば、 `through`で指定して**多対多**の関係をつくって`Active Record モデル`として扱うことができる。
  <br>👉 *「コト」を`RESTリソース`とした捉える* ⇨ *`CRUD操作`として表現できる*
