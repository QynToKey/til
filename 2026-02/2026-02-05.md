## JavaScript (TIPS)

  ### 文章の表示・非表示を切り替える

  - `=`は代入、比較には`===`を使う *👈 ウッカリしがち*
  - `if 文`は**イベントリスナーの内側**で❗️
    - `if` / `else` は「評価された瞬間の状態」で分岐する
      <br> 👉 *状態が変わる処理（クリックなど）の中で判定する必要がある*
    - イベントリスナーの登録は１回だけにすべき
  - 表示状態の判定はテキストではなく **DOM の状態**(`class`)を見る方が安全
      <br> 👉 *テキストは「表示用」、class は「状態そのもの」だから*

    #### 実装例
    - `classList.contains()`
      <br> クラスを「持っているか」を `boolean` で判定できる
    - `classList.toggle()`
      <br> クラスの付け外しを1行で書ける
    - `contains` + `toggle` + `三項演算子` の組み合わせでシンプルな**トグル処理**が書ける

    ```bash
    const isClosed = contents.classList.contains('hidden');
    contents.classList.toggle('hidden');
    btn.textContent = isClosed ? '閉じる' : 'もっと読む';
    ```

      - 用語と概念の整理
      | 見た目      | 種類    | 意味    |
      | -------- | ----- | ----- |
      | `.xxx`   | プロパティ | 値・状態  |
      | `.xxx()` | メソッド  | 処理・動作 |

        - `classList` は **プロパティ**
        - `contains()` / `toggle()` は **メソッド**
        - `() `が付くものは「何かをする」

  ### 表示モードを切り替える

  - 「**クラス**」と「**セレクター**」について、もう一度整理する必要がありそう。
  <br>👉 *CSSが複雑な構成になると、それをJSで引き取らなくちゃと複雑に考えすぎてしまうのだが、そもそもJSが扱うのは「クラス」だけで、「セレクター」の構造はCSSの責務*

    #### `クラス` と `セレクター`

    | 観点| クラス | セレクター |
    | --- | --- | --- |
    | 実体      | `DOM` に**存在する** | **存在しない**（条件式） |
    | `JS` で操作  | ⭕️ | ❌ |
    | `CSS` で使う | ⭕️ | それ自体が **CSS ** |
    | 例 | `hidden`  | `button i.hidden` |

    #### `CSS` と `JS ` の責務分担

    | 場所| 役割 |
    | --- | --- |
    | CSS セレクター | 「どの要素にスタイルを当てるか」 |
    | JS classList | 「その要素がどの class を持つか」 |

    ```bash
    # CSS で「関係性」を書く
      body.dark {
        background: #111;
        color: #fff;
      }

      body.dark h2 {
        color: #ffcc00;
      }

    # JS では「状態」をトグル
      btn.addEventListener('click', () => {
        body.classList.toggle('dark');
        light.classList.toggle('hidden');
        dark.classList.toggle('hidden');
      });
    ```

  #### 学んだこと

  - CSS の**ネスト**や、**結合子**、**擬似クラス**などに惑わされない❗️

    <br>👉 *JS は、クラス（状態）を付ける／外す*
    <br>👉 *CSS は、セレクターで「要素」を拾って見た目を整える*
