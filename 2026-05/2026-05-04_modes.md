# [co-READER](https://github.com/QynToKey/co_reader)（day: 26_2）： 孤読 / 共読モード

## 0️⃣ 現状と実装方針

> 現状

`Membership` モデルには既に `mode enum（solo: 0, co_reading: 1）` が存在するが、表示・操作への反映が未実装。

> 実装方針

- モード切り替えボタン + DB 永続化（ページ遷移後も維持）
- 孤読モード時に他ユーザーのアノテーションを非表示にする（オプションで表示切り替え可）
- 共読モード時は新規書き込みUIを非活性化する（コメントは許可）

## 2️⃣ `ReadingSessionsController` に `update_mode` アクションを追加

- 既存の `before_action` はすでに `require_login` のみ。
- `update_mode` アクションを追加（`set_reading_session` は `inline` で行う）:

```ruby
# app/controllers/reading_sessions_controller.rb
def update_mode
  reading_session = ReadingSession.find(params[:id])
  unless reading_session.members.include?(current_user)
    redirect_to reading_sessions_path, alert: "参加していないセッションです"
    return
  end
  membership = reading_session.memberships.find_by(user: current_user)
  membership.update!(mode: membership.solo? ? :co_reading : :solo)
  redirect_to reading_session_text_path(reading_session, params[:text_id]),
              notice: membership.co_reading? ? "共読モードに切り替えました" : "孤読モードに切り替えました"
end
```

### `annotationsValueChanged`** を以下に差し替え

```js
// app/javascript/controllers/text_selection_controller.js
annotationsValueChanged() {
  if (this.annotationsValue.length === 0) return
  const seen = new Set()
  this.annotationsValue.forEach(a => {
    const key = `${a.annotation_type}:${a.start_position}:${a.end_position}`
    if (seen.has(key)) return
    seen.add(key)
    const span = this.#renderAnnotation(a)
    if (span && !a.is_own) {
      span.dataset.readonly = "true"
      if (this.modeValue === "solo") span.classList.add("d-none")
    }
  })
}
```

### `toggleOtherAnnotations()` メソッド を追加

```js
// app/javascript/controllers/text_selection_controller.js
toggleOtherAnnotations() {
  const spans = this.bodyTarget.querySelectorAll("[data-readonly='true']")
  if (spans.length === 0) return
  const isHidden = spans[0].classList.contains("d-none")
  spans.forEach(span => span.classList.toggle("d-none", !isHidden))
  if (this.hasOtherAnnotationsToggleTarget) {
    this.otherAnnotationsToggleTarget.textContent = isHidden
      ? "他のアノテーションを非表示 ▲"
      : "他のアノテーションを表示 ▼"
  }
}
```

📝 今回追加したロジックの流れ

| 場面 | 動作 |
| --- | --- |
| ページロード | `annotationsValueChanged` が自分 + 他ユーザーの全アノテーションを描画。他ユーザー分には `data-readonly="true"` を付け、孤読モードなら `d-none` で隠す |
| ActionCable 受信 | 孤読モードかつ `user_id` が自分以外 → `return` でスキップ（リアルタイム書き込みも非表示） |
| トグルボタン | `[data-readonly='true']` の span を一括で `d-none` トグル |
| テキスト選択（共読） | `#handleMouseup` 先頭で `return` → ツールバーが表示されない |
| アノテーションクリック（共読） | `#showCoReadingPopup` で type/color/variant/delete ボタンを隠してからポップアップ表示 |
| ポップアップ閉じる | `#hideEditPopup` が `[data-popup-action]` 全要素の `d-none` を解除してリセット |

### 動作確認

-[ ] テキストページでモード切り替えボタンをクリックし、ページが再読み込みされてモードが変わること
-[ ] ページを遷移し戻ってきても、切り替えたモードが維持されていること（DB 永続化）
-[ ] 孤読モード: 自分のアノテーションのみ表示、トグルで他ユーザーのアノテーションを表示/非表示できること
-[ ] 孤読モード: 他ユーザーがリアルタイムでアノテーションを作成しても自分の画面に表示されないこと
-[ ] 共読モード: テキスト選択してもツールバーが表示されないこと
-[ ] 共読モード: 既存アノテーションをクリックするとポップアップが開き、コメントフォームのみ操作できること（type/color/variant/delete ボタンが非表示）
-[ ] 共読モード: コメント送信が正常に動作すること
-[ ] 共読モード: 他ユーザーのアノテーションがリアルタイムに表示されること
