## 学習プランの再設計や、環境構築など。

### やったこと

- GitHubリポジトリを`clone`

- `Gemfile`の内容を確認
  - Ruby / Rails / DB バージョンを確定
  - `Dockerfile` / `docker-compose.yml` を作成
    ```
    docker compose build
    ↓
    docker compose up -d
    ↓
    http://localhost:3000 で Railsの起動を確認
    ```

### 学んだこと

- Railsは`Gemfile`のあるディレクトリがすべての基準
- `git clone`はプロジェクトの「箱」を作る
  🤛 *親ディレクトリ名は本質ではない(任意で構わない)*
- Dockerは カレントディレクトリ`.`基準で動く


  #### 構成メモ
    ```
    Ruby: 3.2.9
    Rails: 7.0.4.3
    DB: sqlite3
    ```

  #### Dockerコマンドの覚書
    ```
    docker compose logs web        # Railsログ確認
    docker compose exec web bash   # コンテナに入る
    docker compose exec web bin/rails # Rails操作
    docker compose exec web bin/dev   # 開発用起動（ある場合）
    ```
