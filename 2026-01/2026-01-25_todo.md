## JavaScriptで「[Todo管理アプリ](https://github.com/QynToKey/MyTodoApp)」をつくる

### CSS tips

- フレックスボックスの`align-items`プロパティの初期値は`stretch`になっている。
  ```
  # 要素が上方に配置されるように`flex-start`を指定
    li {
      display: flex;
      justify-content: space-between;
      align-items: flex-start;
    }
  ```

- **子孫結合**と**隣接結合**
  ```
  # li要素の中にある →input要素が:checked 状態で →その直後に隣接しているspan 要素
    li input:checked + span {
      color: #aaa;
      text-decoration: line-through;
    }

  # ネスト構文
    li {
      …

      input:checked + span {
        color: #aaa;
        text-decoration: line-through;
      }
    }
  ```

- **ネスト**は*構造を表現するため*に使うべき。

  #### ネストを使うときのチェックリスト
  ```
  □ そのネストは「文脈の共有」になっているか？

    └ 可読性はどうか

  □ そのネストを外したら「意味の塊」が壊れるか？

    └ 意味構造を表現しているか

  □ そのネストは「構造を固定」しすぎていないか？

    └ 将来の変更耐性を考慮する

  □ そのネストの「深さ」は適切か？

    └ 深さ３以上になると設計負債になりやすい
  ```

### 実装について

- `li`要素を動的に描画する
  ```
  # ここでは「関数式」で記述
   const renderTodo = (todo) => {

    # HTMLの構造をコメントでメモしておくと、確認しながら作業を進められる。
      /*
      - li
        - label
          - input
          - span
        - button
      */

      # DOM構造の内側から組み上げていく
      const input = document.createElement('input');
      input.type = 'checkbox';
      input.checked = todo.isCompleted;
      const span = document.createElement('span');
      span.textContent = todo.title;
      const label = document.createElement('label');
      label.appendChild(input);
      label.appendChild(span);
      const btn = document.createElement('button');
      btn.textContent = 'x';
      const li = document.createElement('li');
      li.appendChild(label);
      li.appendChild(btn);
      document.querySelector('#todos').appendChild(li);
    };
  ```

  - **関数宣言**と**関数式**のちがい

    - 関数宣言 → 巻き上げがある（どこに書いても動作する）
    - 関数式 → 宣言したあとでしか使えない
