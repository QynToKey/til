## 1️⃣ TOP ページ他の仕上げ

### 日付から当該日の学習ログ (詳細画面) へ遷移する

「カレンダー機能」の実装は MVP 後に先送りしたが、初期構想段階にあった最低限の機能は実装しておきたい。

#### □ `index` ページの「日付」に「一覧ページ」へのリンクを追加

```ruby
# app/views/learning_records/index.html.erb
<td><%= link_to record.study_date.strftime('%Y-%m-%d'), learning_records_path(date: record.study_date) %></td>
```

👉 *`learning_records_path()` に `record.study_date` を渡す*

#### □ `LearningRecordsController` の `index` アクションに条件分岐を追加

```ruby
# app/controllers/learning_records_controller.rb
  def index
    ・・・

    if params[:tag_id].present?
      # ユーザーが所有するタグの中から、指定されたタグIDに該当するタグを見つける
      @current_tag = current_user.tags.find(params[:tag_id])
      # タグIDが指定されている場合は、そのタグが付いている学習記録のみを表示する
      @learning_records = @learning_records.joins(:tags).where(tags: { id: @current_tag.id })
    elsif params[:date].present?
      # 学習日が指定されている場合は、その日に該当する学習記録のみを表示する
      @learning_records = @learning_records.where(study_date: params[:date])
    end
  end
```

#### □ `user.rb` に `total_learning_minutes_by_date()` メソッドを追加

```ruby
  # 指定した日付の学習時間の合計を計算する
  def total_learning_minutes_by_date(date)
    learning_records.where(study_date: date).sum(:duration_minutes)
  end
```

#### □ `index` ページの絞り込み表示を修正

```erb
<%# 絞り込み条件によって見出しを変える %>
<% if params[:tag_id].present? %>
  <h1>'<%= @current_tag.name %>' の学習ログ</h1>
<% elsif params[:date].present? %>
  <h1>'<%= Date.parse(params[:date]).strftime('%Y/%m/%d') %>' の学習ログ</h1>
<% else %>
  <h1>学習ログ</h1>
<% end %>
```

```erb
<%# 日付で絞り込んでいる場合は全件表示へのリンクを表示する %>
<% elsif params[:date].present? %>
  <p>'<%= Date.parse(params[:date]).strftime('%Y/%m/%d') %>' の学習時間累計： <%= (current_user.total_learning_minutes_by_date(Date.parse(params[:date])) / 60.0).round(1) %> 時間</p>
  <%= link_to "全件表示へ戻る", learning_records_path %>
```
