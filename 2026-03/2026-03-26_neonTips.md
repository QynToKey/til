# NEON tips

## 本番環境の DB を簡易バックアップする

> ローカルで手動で行う方法

### 0️⃣ PostgreSQL のインストールを確認

```bash
pg_dump --version
```

👉 *エラーが出たら未インストール*

```bash
# Mac（Homebrew）の場合
brew install postgresql
```

---

### 1️⃣ ターミナルでコマンドを叩く

👉 *ローカルの `backups/backup.sql` にバックアップ*

```bash
pg_dump "DATABASE_URL" > backups/backup.sql
```

※ `DATABASE_URL` は Neon のダッシュボードから取得する（Connection string）

  ⬇️

実行後、`backup.sql` が作成されるので `.gitignore` に追加

```ruby
# .gitignore
backup.sql
```

---

### 2️⃣ コマンドをカスタマイズする

#### バックアップファイルに日付を添える

```bash
pg_dump "DATABASE_URL" > backups/backup_$(date +%F).sql
```

```ruby
# .gitignore
backup_*.sql
```

#### コマンドをエイリアス化する

```bash
backupdb() {
  mkdir -p backups
  pg_dump "DATABASE_URL" > backups/backup_$(date +%F).sql
  echo "backup done: $(date)"
}
```

👉 *次から `backupdb` でバックアップが実行される*
👉 *実行後にログが流れる*

⚠️ セッションを閉じると関数定義は無効化される *（永続化したい場合は `~/.zshrc` に関数を記述する）*

---

### 3️⃣ 古いバックアップを消す

```bash
find . -name "backup_*.sql" -mtime +7 -delete
```

👉 *日付つきファイルで管理する場合は、一定期間ごとに不要ファイルを整理しておきたい（上記は「7日より古いファイルを削除」する例）*

---

### 4️⃣ バックアップから復元する

```bash
# 2026-03-26 のデータを復元する場合
psql "DATABASE_URL" < backup_2026-03-26.sql
```

⚠️ バックアップからの復元は既存データを上書きするので要注意

対策：

- 本番 DB に直接実行しない❗️
  👉 *必ず別環境（NEON に新しいブランチをつくるなど）で確認してから実行する*
- ローカル DB に復元して確認
  👉 *（ローカルに PostgreSQL が必須❗️）*

---

### 5️⃣ バックアップ確認

```bash
ls -lh backup_*.sql
```

👉 *ファイルが生成されていれば成功*
