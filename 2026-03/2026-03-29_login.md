```ruby

```

```ruby
Rails.application.config.sorcery.submodules = [:remember_me]
```

## 3️⃣ コントローラーの修正

👉 *`login()` の第3引数に `params[:remember_me]` を渡す*

```ruby
# app/controllers/user_sessions_controller.rb
```

この実装で、

- チェックボックスがオンなら `1`、オフなら `nil` が渡される。
- Sorcery が自動的に判断して `remember_me!` を呼んでくれる。

📝 Sorcery の `login` メソッドの定義

```ruby
def login(username, password, remember_me = false)
```

- 第1引数 → ユーザー名（email）
- 第2引数 → パスワード
- 第3引数 → `remember_me`（省略可能、デフォルトは `false`）

## 4️⃣ ビューにチェックボックスを追加

```erb
<%# app/views/user_sessions/new.html.erb %>
  <div class="mb-3 form-check">
    <%= check_box_tag :remember_me, "1", false, class: "form-check-input" %>
    <%= label_tag :remember_me, "ログイン状態を保持する", class: "form-check-label" %>
  </div>
```

📝 `<%= check_box_tag :remember_me, "1", false, class: "form-check-input" %>`

| 引数 | はたらき |
| --- | --- |
| `:remember_me` | フォームのパラメーター名。チェックされたとき `params[:remember_me]` としてコントローラーに渡される。 |
| `"1"` | チェックされたときに送信される値。`params[:remember_me]` の値が `"1"` になる。Sorceryは `"1"` を truthy として扱い `remember_me!` を呼び出す。 |
| `false` | チェックボックスの初期状態。`false` なので最初はチェックなし、`true` にすると最初からチェック済みになる。 |
| `class: "form-check-input"` | HTMLの属性。Bootstrapのスタイルを適用している。 |
